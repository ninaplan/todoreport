## Notion 토큰 끊김 관련 조사 기록 (2026-06-24)

### 결론
- Notion access_token에는 시간 기반 만료가 없음 (Notion 정책 확인)
- 토큰이 끊기는 진짜 원인: 같은 Notion 워크스페이스를 다른 디바이스/빌드에서
  다시 OAuth 인증(재연결)하면, Notion이 기존 access_token과 refresh_token을
  "동시에" 무효화함
- curl로 재현 테스트 시 invalid_grant 에러로 확인됨
- "워크스페이스 1개 = 플래너 1개" 원칙이 사실상 강제되는 이유가 이것

### 처리 방향
- 이미 구현해둔 refresh token 인프라(저장/전파)는 롤백하지 않고 유지
  - 이유: 향후 Notion이 시간 기반 만료 정책을 도입하거나 토큰 정책이
    바뀔 경우를 대비한 자산으로 판단
  - 현재 버그(워크스페이스 중복 연결)는 refresh로 해결되지 않음 — 별개 문제
- 관련 코드에는 "원인은 시간 만료가 아니라 중복 OAuth"라는 설명 주석을
  추가함 (2026-06-26, 백엔드 lib/notion-auth.ts, refresh route,
  iOS PlannerItem.swift)

### 향후 검토 항목
- refresh token 인프라의 실질적 효용 재평가 — 코드가 6개 파일에 분산되어
  있어 유지보수 부담이 있음. 당장 제거하지는 않되, 추후 사용 사례가
  명확해지지 않으면 정리 검토
- 워크스페이스 중복 연결 자체를 막는 방어 로직 (아직 미설계) —
  예: 같은 workspace_id로 재연결 시도 시 경고/차단
- "워크스페이스 1개 = 플래너 1개" 원칙과 멀티유저 공유 기능(V2)의
  아키텍처 충돌 — 별도 설계 결정 필요
